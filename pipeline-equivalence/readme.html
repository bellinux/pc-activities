Guía de Análisis de Equivalencia (Microbit vs Protobject)

Este documento explica paso a paso cómo un humano o un agente autónomo puede agregar una nueva actividad al sistema y generar automáticamente todo el análisis comparativo, desde la extracción de texto hasta la generación de grafos y métricas algorítmicas.

El sistema es determinístico en su ejecución (scripts de Python), pero depende de un LLM (Large Language Model) para la fase de abstracción semántica inicial (creación de los JSONs).

🤖 ATENCIÓN AGENTE / HUMANO: ¡ESTUDIA LOS EJEMPLOS!

Antes de generar cualquier JSON, ES OBLIGATORIO que revises cómo están estructurados los JSONs de la actividad de ejemplo que ya existe en el sistema.

Abre la carpeta reporte/actividades/vela/ y analiza detenidamente la estructura, llaves y jerarquías de:

  • text_makecode.json / text_protobject.json (Fíjate en cómo se cuenta la indentación).
  • ast_makecode.json / ast_protobject.json (Fíjate en el árbol jerárquico children).
  • pdg_makecode.json / pdg_protobject.json (Fíjate en las listas de nodes y edges).
  • tabla_ir.json (Fíjate en el mapeo ontológico cruzado).

Deberás replicar exactamente este mismo esquema y nombres de llaves para la nueva actividad, o de lo contrario el pipeline de Python y los visores fallarán catastróficamente.

Flujo de Trabajo (Workflow)

Sigue estos pasos en estricto orden para agregar una nueva actividad (ej. mi_actividad):

Paso 1: Preparar el Entorno

  1. Dirígete a la carpeta reporte/actividades/.
  2. Crea una nueva carpeta con el nombre de tu actividad (por ejemplo: reporte/actividades/mi_actividad).
  3. Guarda dentro de esta carpeta las dos imágenes originales del código:
    • microbit.png (El código fuente en MakeCode/Microbit)
    • protobject.png (El código fuente en Protobject)

Paso 2: Extracción de Texto (LLM)

Usa el LLM para leer el texto de las imágenes. Pide al modelo que genere un JSON extrayendo línea por línea (preservando la indentación mediante espacios).

  1. Crea text_makecode.json extrayendo el texto de microbit.png.
  2. Crea text_protobject.json extrayendo el texto de protobject.png.
Nota: La estructura del JSON debe contener un arreglo de strings bajo la clave "extracted_text".

Paso 3: Generación del Árbol de Sintaxis Abstracta - AST (LLM)

Proporciona los textos extraídos al LLM y pide que genere la estructura lógica jerárquica (AST).

  1. Crea ast_makecode.json basado en el texto de Microbit.
  2. Crea ast_protobject.json basado en el texto de Protobject.
Nota: La estructura debe incluir anidamiento mediante la propiedad "children".

Paso 4: Generación del Grafo de Dependencia - PDG (LLM)

Pide al LLM que analice los ASTs y determine las dependencias de control y de datos (edges) entre los diferentes nodos, construyendo así el Program Dependence Graph.

  1. Crea pdg_makecode.json.
  2. Crea pdg_protobject.json.
Nota: Debe contener una lista de "nodes" (con IDs) y "edges" (origen, destino, tipo).

Paso 5: Generación de Tabla Ontológica IR (LLM)

Usa el LLM para mapear la equivalencia conceptual entre los bloques de ambas plataformas.

  1. Crea tabla_ir.json en la misma carpeta. Este JSON debe tener un arreglo de objetos que relacionen un "canonical_node" con el "makecode_block" y el "protobject_block".

Paso 6: Ejecución del Pipeline Algorítmico (Python)

Una vez que todos los JSONs han sido generados, el resto del proceso matemático es automático.

  1. Abre tu terminal.
  2. Navega a la carpeta de scripts: cd reporte/scripts/
  3. Ejecuta el pipeline pasando el nombre de la carpeta que creaste en el Paso 1:
python run_pipeline.py mi_actividad

¿Qué hace este script?

  • Lee los archivos pdg_makecode.json y pdg_protobject.json.
  • Construye grafos matemáticos usando NetworkX.
  • Exporta los grafos visuales en formato vectorial: pdg_makecode.svg y pdg_protobject.svg (usando Graphviz).
  • Calcula métricas topológicas (Graph Edit Distance e Isomorfismo VF2).
  • Guarda los resultados automáticamente en metricas.json.

Paso 7: Actualizar el Dashboard Principal (index.html)

Para que los resultados sean visibles en la plataforma web:

  1. Abre el archivo raíz index.html.
  2. Busca el bloque HTML de una actividad existente (por ejemplo, el bloque <div id="act-vela"> ... </div>).
  3. Copia y pega todo ese bloque justo debajo.
  4. Reemplaza todas las apariciones de la palabra vela por el nombre de tu nueva carpeta (ej. mi_actividad).
  5. Guarda el archivo.

Paso 8: Visualización y Verificación final

  1. Asegúrate de tener el servidor web corriendo desde la raíz del proyecto:
    2. python -m http.server 8080
  2. Abre http://localhost:8080 en tu navegador.
  3. Haz clic en los diferentes visores (Extracción, AST, Tabla IR y PDG Interactivo) para la nueva actividad y verifica que todo se renderice correctamente.
